Auric Network

Zero-Proxy Network System for Minecraft servers.

No BungeeCord · No Velocity · Native Transfers

AuricNetwork allows you to create a full Minecraft server network without any proxy (BungeeCord, Velocity). It uses Paper's native Player#transfer method (1.20.5+) to move players directly between servers.

Download on Modrinth Discord Support YouTube Channel

Important: On the target server, you must set accepts-transfers=true in its server.properties file for the transfers to work.

Reference

Commands

The complete list of commands available in AuricNetwork. Admin commands are prefixed with /auricnetwork; player-facing commands like /server, /spawn and /lobby are also available.

/auricnetwork reload
Reloads config and language files. Permission: auricnetwork.admin
/auricnetwork menu
Opens the server selector GUI. Permission: auricnetwork.menu
/auricnetwork setjoinspawn
Sets the join spawn at your current location. Permission: auricnetwork.admin
/auricnetwork setrespawnspawn
Sets the respawn spawn at your current location. Permission: auricnetwork.admin
/server <name>
Transfers you to another server. Permission: auricnetwork.server
/spawn [join|respawn]
Teleports to the configured spawn point. Permission: auricnetwork.spawn
/lobby or /hub
Sends you to the lobby (supports random balancing). Permission: auricnetwork.lobby

Setup

Configuration Guide

The config.yml file controls every aspect of AuricNetwork. Below is a detailed explanation of each section.

Topics Servers Transfer Immersion Language Join & Respawn Spawn Lobby Balancing Server Status API

1 Servers

List all servers in your network. The key is the name used in commands and the menu, the value is the IP:port of each server.

View example
YAML 
servers:
  lobby: "127.0.0.1:25565"
  survival: "127.0.0.1:25566"

2 Transfer Immersion

Delay before transfer (in seconds), sound effect, and volume fade-out. These options add a small cinematic moment when a player is moved between servers.

View example
YAML 
transfer_immersion:
  delay: 3.0
  sound:
    enabled: true
    sound_id: "block.amethyst_block.hit"
    fade_out: true

3 Language

Set en (English) or es (Spanish). Leave empty to auto-detect from the player's client locale.

View example
YAML 
language: "en"

4 Join & Respawn Spawn

Enable and set coordinates for where players spawn when they join or respawn. Each section supports a world name, position, and rotation.

View example
YAML 
join_spawn:
  enabled: false
  world: "world"
  x: 0.5
  y: 64.0
  z: 0.5
  yaw: 0.0
  pitch: 0.0

respawn_spawn:
  enabled: false
  world: "world"
  x: 0.5
  y: 64.0
  z: 0.5
  yaw: 0.0
  pitch: 0.0

5 Lobby Balancing

If enabled, /lobby will send the player to a random server from the list instead of the default lobby.

View example
YAML 
lobby:
  enabled: false
  servers:
    - "lobby1"
    - "lobby2"

6 Server Status API

Fetches live player counts and online status from api.mcsrvstat.us. The update interval is configured in seconds.

View example
YAML 
server_status_api:
  enabled: false
  update_interval_seconds: 60

Fallback

Rescue Server

When a server shuts down gracefully (e.g., /stop or a restart), AuricNetwork automatically transfers all online players to a configured rescue server. This prevents disconnections and improves user experience.

The fallback logs a detailed message to the console:

======================================== [AuricNetwork] Rescue server active! Transferred 12 players to lobby (127.0.0.1:25565) Server is shutting down gracefully. ========================================

To enable it, add the following to your config.yml:

YAML 
rescue_server:
  enabled: true
  target: "lobby"

Note: Works only on controlled shutdowns (not on crashes or kill -9).
